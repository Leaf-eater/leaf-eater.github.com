并查集系列（一）——Social network connectivity

发表于 2017-02-02 | 分类于 算法

　　course并查集的课后题之一，看完之后没有开始没有什么思路，也没找到什么答案，想了想写了一下，感觉受益挺多的

##问题描述：
　　Social network connectivity. Given a social network containing n members and a log file containing m timestamps at which times pairs of members formed friendships, design an algorithm to determine the earliest time at which all members are connected (i.e., every member is a friend of a friend of a friend … of a friend). Assume that the log file is sorted by timestamp and that friendship is an equivalence relation. The running time of your algorithm should be mlogn or better and use extra space proportional to n.
解题思路：
　　首先对于这样的问题我们肯定要使用并查集来求解，题目求是在散列刚刚全部连通的时候的状态，所以我们需要判断什么时候刚刚连通。这时候会想到，全部连通时只会有一个父节点的存在所以，当我们做一次连接操作的时候做一次父节点的判断就行了，当使用weight quick union 的查询父节点操作的时间复杂度为logn，总的复杂度m(logn+1)=mlogn所以可解，下面附上java实现代码，希望可以帮助和我一样的初学者。

如何在基于的hexo博客中插入图片

发表于 2017-02-01 | 分类于 玩玩小东西

##如何在基于hexo的博客中插入图片
在写FirstBlog中因为牵扯要插入以上图片嘛，开始的想法很简单file\\\D:\blog\picture\需要的图片 这样(也许是一时脑抽。。现在看来还真是不可理喻)。好吧，果然部署不上去，之后老老实实的google…吐槽完毕，下面是正文：
1.编辑你的配置文档_config.yml，找到并更改设置为post_asset_folder: true。这个设置的意思是设置一种模式：当你新建了一个名为 newBlog 的文章时，会自动创建一个名为 newBlog 的资源文件夹，这个文件夹用于存放这边博客所需的资源文件比如我们要插入的图片。
2.在hexo的目录下执行命令npm install https://github.com/CodeFalling/hexo-asset-image --save 。在输入前请确认你已经安装了npm。这里安装了一个github上的一个工具，用于方便的调用本地图片(github地址)。
3.接下来就是使用啦，这里可以使用markdown的正常语法调用图片![图片配字](newBlog/需要调用的图片.jpg)，这时候你就可以看到图片正常显示啦~。

网上的说法很简单，将你的文章同步到对应的github Page需要这么几步：
1.写一篇博客 (new BlogName)
2.执行命令 hexo g
3.执行命令 hexo s
4.访问localhost:4000,确认没问题之后执行命令 hexo d 完成同步
这里解释一下这里的各个命令，所谓知其所以然：(更多hexo相关的配置等一系列问题请看这个地址)

所以以上指令对应就对应:
生成(generate)->开启本地服务(server)->部署(develop)
明白了这些操作的原理就可以更方便我们的操作，比如：server指令只是在开启本地服务，所以如果确定自己写的没问特其实不写也可以的~~
